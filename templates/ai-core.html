Data Flow Pipeline

Every user request follows a strict security pipeline ensuring data is checked properly.

  1. User Input: User enters data into the Form or uploads a file.
  2. Preparation: The system converts this data into a JSON format (e.g., {protocol: "TCP"}).
  3. Transfer: Data is sent to the Backend Server.
  4. Routing: The server sends the data to the correct tool script.
  5. Processing: The tool (AI or Rule Engine) analyzes the data and finds results.
  6. Report: Results are displayed back to the user on the dashboard.
